## From .pyc to python code

Za **.pyc** podatke ("kompilirani" python) trebali biste poƒçeti poku≈°avati da **izvuƒçete** **originalni** **python** **kod**:
```bash
uncompyle6 binary.pyc  > decompiled.py
```
**Budite sigurni** da binarni fajl ima **ekstenziju** "**.pyc**" (ako ne, uncompyle6 neƒáe raditi)

Dok izvr≈°avate **uncompyle6**, mogli biste naiƒái na **sledeƒáe gre≈°ke**:

### Gre≈°ka: Nepoznat magiƒçni broj 227
```bash
/kali/.local/bin/uncompyle6 /tmp/binary.pyc
Unknown magic number 227 in /tmp/binary.pyc
```
Da biste to popravili, potrebno je da **dodate ispravan magiƒçni broj** na poƒçetku generisanog fajla.

**Magiƒçni brojevi se razlikuju u zavisnosti od verzije pythona**, da biste dobili magiƒçni broj za **python 3.8**, potrebno je da **otvorite python 3.8** terminal i izvr≈°ite:
```
>> import imp
>> imp.get_magic().hex()
'550d0d0a'
```
The **magic number** in this case for python3.8 is **`0x550d0d0a`**, then, to fix this error you will need to **add** at the **beginning** of the **.pyc file** the following bytes: `0x0d550a0d000000000000000000000000`

**Jednom** kada ste **dodali** taj magiƒçni zaglavlje, **gre≈°ka bi trebala biti ispravljena.**

This is how a correctly added **.pyc python3.8 magic header** will look like:
```bash
hexdump 'binary.pyc' | head
0000000 0d55 0a0d 0000 0000 0000 0000 0000 0000
0000010 00e3 0000 0000 0000 0000 0000 0000 0000
0000020 0700 0000 4000 0000 7300 0132 0000 0064
0000030 0164 006c 005a 0064 0164 016c 015a 0064
```
### Gre≈°ka: Decompiling generic errors

**Druge gre≈°ke** kao ≈°to su: `class 'AssertionError'>; co_code should be one of the types (<class 'str'>, <class 'bytes'>, <class 'list'>, <class 'tuple'>); is type <class 'NoneType'>` mogu se pojaviti.

To verovatno znaƒçi da **niste ispravno dodali** magiƒçni broj ili da niste **koristili** **ispravan magiƒçni broj**, pa se **pobrinite da koristite ispravan** (ili poku≈°ajte novi).

Proverite dokumentaciju o prethodnim gre≈°kama.

## Automatski alat

[**python-exe-unpacker alat**](https://github.com/countercept/python-exe-unpacker) slu≈æi kao kombinacija nekoliko alata dostupnih u zajednici, dizajniranih da pomognu istra≈æivaƒçima u raspakivanju i decompiling izvr≈°nih datoteka napisanih u Pythonu, posebno onih kreiranih sa py2exe i pyinstaller. Ukljuƒçuje YARA pravila za identifikaciju da li je izvr≈°na datoteka zasnovana na Pythonu i potvrƒëuje alat za kreiranje.

### ImportError: Ime datoteke: 'unpacked/malware\_3.exe/**pycache**/archive.cpython-35.pyc' ne postoji

Uobiƒçajen problem koji se javlja ukljuƒçuje nepotpunu Python bytecode datoteku koja je rezultat **procesa raspakivanja sa unpy2exe ili pyinstxtractor**, koja zatim **ne mo≈æe biti prepoznata od strane uncompyle6 zbog nedostatka broja verzije Python bytecode-a**. Da bi se to re≈°ilo, dodata je opcija za dodavanje, koja dodaje neophodan broj verzije Python bytecode-a, olak≈°avajuƒái proces decompiling-a.

Primer problema:

## Analyzing python assembly

Ako niste mogli da izvuƒçete python "original" kod prateƒái prethodne korake, onda mo≈æete poku≈°ati da **izvuƒçete** **assembly** (ali **nije ba≈° opisno**, pa **poku≈°ajte** da ponovo **izvuƒçete** originalni kod). U [ovde](https://bits.theorem.co/protecting-a-python-codebase/) sam prona≈°ao vrlo jednostavan kod za **disassemble** _.pyc_ binarni (sreƒáno sa razumevanjem toka koda). Ako je _.pyc_ iz python2, koristite python2:
```bash
>>> import dis
>>> import marshal
>>> import struct
>>> import imp
>>>
>>> with open('hello.pyc', 'r') as f:  # Read the binary file
...     magic = f.read(4)
...     timestamp = f.read(4)
...     code = f.read()
...
>>>
>>> # Unpack the structured content and un-marshal the code
>>> magic = struct.unpack('<H', magic[:2])
>>> timestamp = struct.unpack('<I', timestamp)
>>> code = marshal.loads(code)
>>> magic, timestamp, code
((62211,), (1425911959,), <code object <module> at 0x7fd54f90d5b0, file "hello.py", line 1>)
>>>
>>> # Verify if the magic number corresponds with the current python version
>>> struct.unpack('<H', imp.get_magic()[:2]) == magic
True
>>>
>>> # Disassemble the code object
>>> dis.disassemble(code)
1           0 LOAD_CONST               0 (<code object hello_world at 0x7f31b7240eb0, file "hello.py", line 1>)
3 MAKE_FUNCTION            0
6 STORE_NAME               0 (hello_world)
9 LOAD_CONST               1 (None)
12 RETURN_VALUE
>>>
>>> # Also disassemble that const being loaded (our function)
>>> dis.disassemble(code.co_consts[0])
2           0 LOAD_CONST               1 ('Hello  {0}')
3 LOAD_ATTR                0 (format)
6 LOAD_FAST                0 (name)
9 CALL_FUNCTION            1
12 PRINT_ITEM
13 PRINT_NEWLINE
14 LOAD_CONST               0 (None)
17 RETURN_VALUE
```
## Python to Executable

Da poƒçnemo, poka≈æaƒáemo vam kako se payloadi mogu kompajlirati u py2exe i PyInstaller.

### Da kreirate payload koristeƒái py2exe:

1. Instalirajte py2exe paket sa [http://www.py2exe.org/](http://www.py2exe.org)
2. Za payload (u ovom sluƒçaju, nazvaƒáemo ga hello.py), koristite skriptu kao ≈°to je prikazano na Slici 1. Opcija ‚Äúbundle\_files‚Äù sa vredno≈°ƒáu 1 ƒáe sve spojiti, ukljuƒçujuƒái Python interpreter, u jedan exe.
3. Kada je skripta spremna, izdaƒáemo komandu ‚Äúpython setup.py py2exe‚Äù. Ovo ƒáe kreirati izvr≈°ni fajl, ba≈° kao na Slici 2.
```python
from distutils.core import setup
import py2exe, sys, os

sys.argv.append('py2exe')

setup(
options = {'py2exe': {'bundle_files': 1}},
#windows = [{'script': "hello.py"}],
console = [{'script': "hello.py"}],
zipfile = None,
)
```
