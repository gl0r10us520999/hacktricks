### 簡単な説明 <a href="#brief-description" id="brief-description"></a>

このチャレンジは、ページ[https://challenge-0521.intigriti.io/captcha.php](https://challenge-0521.intigriti.io/captcha.php)にあるXSSに脆弱なフォームを提供しています。\
このフォームは、[https://challenge-0521.intigriti.io/](https://challenge-0521.intigriti.io)にあるiframeを介して読み込まれます。

フォームは、ユーザーの入力をJavaScriptの`eval`関数内に挿入することがわかりました。これは通常、**任意のJavaScriptの実行**につながる可能性があるため、これは良い例です。\
ただし、ユーザーの入力を`eval`関数内に挿入する前に、正規表現`/[a-df-z<>()!\\='"]/gi`でチェックされるため、これらの文字のいずれかが見つかると、ユーザーの入力は`eval`内で実行されません。\
とはいえ、正規表現保護をバイパスして`alert(document.domain)`を実行し、危険な`eval`関数を悪用する方法が見つかりました。

### HTMLへのアクセス <a href="#accessing-the-html" id="accessing-the-html"></a>

文字`e`がユーザー入力として許可されていることがわかりました。また、`id="e"`を使用しているHTLM要素があることもわかりました。したがって、このHTLM要素は、単純に変数`e`を使用してJavaScriptからアクセスできます：\
![](https://i.imgur.com/Slq2Xal.png)

また、JSでは、**オブジェクトの属性にはドットまたは角かっこ内の文字列を使用してアクセス**できます。したがって、`document`オブジェクトの`domain`属性には、次のいずれかの方法でアクセスできます：
```javascript
document.domain
document["domain"]
```
同じことが、関数（メソッド）である属性にも当てはまります：
```javascript
document.write("1")
document["write"]("1")
```
次に、`e` HTML要素から、次のような方法で`document`オブジェクトにアクセスすることが可能です：
```javascript
e["parentNode"]["parentNode"]["parentNode"]["parentNode"]["parentNode"]
```
### JSコードを文字列として使用して関数を括弧なしで呼び出す <a href="#calling-a-function-without-parenthesis-with-js-code-as-string" id="calling-a-function-without-parenthesis-with-js-code-as-string"></a>

オブジェクト`document`から`write`関数を呼び出して、**ブラウザが実行する任意のHTMLテキストを書き込む**ことが可能です。\
ただし、`()`文字は**禁止**されているため、それらを使用して関数を呼び出すことはできません。それでも、**バックティック**（\`\`）を使用して関数を呼び出すことが可能です。\
さらに、`${...}`を使用して実行されるJavaScriptコードを文字列として記述することが可能です。
```javascript
`${"alert(document.location)"}`
```
したがって、このテクニックを使って`document`オブジェクトにアクセスし、括弧なしで関数を実行することで、**アラートを実行する**ことが可能です。
```javascript
e["parentNode"]["parentNode"]["parentNode"]["parentNode"]["parentNode"]["write"]`${"<script>alert(document.location)</script>"}`
```
### 最終的な禁止文字バイパス <a href="#final-forbidden-characters-bypass" id="final-forbidden-characters-bypass"></a>

ただし、まだ1つ問題が残っています。エクスプロイトのほとんどの文字は、正規表現 `/[a-df-z<>()!\\='"]/gi` に現れるため、**禁止**されています。しかし、エクスプロイト内のすべての**禁止文字が文字列であり、エクスプロイト内の文字列でない文字 (e\[]\`${}) は許可**されていることに注意してください。\
これは、**許可された文字から禁止文字を文字列として生成することが可能であれば、エクスプロイトを生成することができる**ということを意味します。\
これを行うために、このチャレンジ向けにカスタムされたアルファベットを生成するための[JSFuck](http://www.jsfuck.com)のようなアルファベットを生成しました。\
**エクスプロイトコード内で完全なアルファベットを見ることができます**（次のサブセクションとファイル_exploit.txt_にあります）。

たとえば、**文字 `a` を生成する**には、**`[[]/e+e][0][1]` にアクセスすることが可能**です。なぜなら、`[[]/e+e][0]` は文字列 `"NaN[object HTMLProgressElement]"` を生成するからです。また、**文字 `f` を生成する**には、**`[][[]]+e][0]` の5番目の文字にアクセスすることが可能**です。なぜなら、この式は文字列 `"undefined[object HTMLProgressElement]"` を生成するからです。\
これらのトリックとさらに複雑なものを使用することで、エクスプロイトに含まれる**すべての文字（文字と記号）を生成することが可能**でした：
```javascript
e["parentNode"]["parentNode"]["parentNode"]["parentNode"]["parentNode"]["write"]`${"<script>alert(document.location)</script>"}`
```
